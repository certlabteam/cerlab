CertLab 업로드·검수

관리자(certlab.team) 계정으로 로그인하세요.

📤 업로드·검수 (자동화용 경량) · 로그아웃
🔎 검증 전용 모드 (?action=validate) — 업로드는 하지 않고 검수만 실행합니다.

📋 결과 로그 id=up_result

모든 처리 결과·안내·차단 사유가 여기에 텍스트로 쌓입니다(자동화가 읽는 곳). 팝업(alert/confirm)은 뜨지 않습니다.

🔎 검수 실행 (QC) window.runQC

업로드하지 않고 파일만 검수합니다. 결과는 아래 #qc_report에 JSON으로, 그리고 window.__QC_RESULT에 담깁니다. 형식: [{section,code,bank,qid,field,idx,sev,desc}]

파일로 검수

기출 JSON(문항 배열 / {questions:[]} / {banks:[...]})을 넣으면 즉시 검수합니다.

Firestore 과목 검수

현재 Firestore에 올라간 해당 과목을 불러와 검수합니다.

🩹 콘텐츠 패치 해설·예시·띄어쓰기 병합

기존 문항의 exp.o / exp.ex / q(띄어쓰기) / opts(띄어쓰기)만 병합. 검증 통과분만 저장 버튼이 뜹니다.

⚡ 레벨업 업로드 adaptive: variantq/calctmpl/…

레벨업·적응형 데이터( {subjects:[…]} 또는 단일 과목 )를 id 기준 병합.

📚 마스터 업로드 개념·표·그래프·암기·인터랙티브

각 마스터를 id 기준 병합(덮어쓰기 아님). 파일별 _meta.generatedAt·레코드별 updatedAt 날짜 게이트 통과 필요.
