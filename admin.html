기출 내보내기 (현재 Firestore → JSON)
자격증을 고르고 다운로드를 누르면 현재 서비스 중인 데이터를 JSON으로 받습니다.
🔍 검수 (위 시험·과목 범위 · 라이브)
내보낸 파일과 함께 이 검수 결과(지적서 TXT)를 데이터방에 넘기면 됩니다. 위에서 고른 시험·과목 범위를 그대로 검사합니다.
※ 이 파일이 현재 Firestore에 올라가 있는 최신 데이터입니다. 데이터방에서 이 파일을 받아 수정한 뒤, 업로드 페이지로 다시 올리면 됩니다.
※ 형식은 업로드와 호환됩니다 (manifest + banks).
기출 업로드 (JSON → Firestore banks)
여기로 JSON 파일을 끌어다 놓거나 클릭해서 선택
통합 번들(내보내기 파일) 또는 과목별 파일(bodybuilding__B.json 등) 모두 가능
⚠️ 실행하면 banks 문서를 덮어씁니다(set/overwrite). manifest는 건드리지 않습니다. 아래 "현재 ver → 새 ver"를 확인 후 실행하세요.
문서 IDcert/subject현재 ver새 ver문항수상태
업로드하면 버전 동기화가 자동으로 실행됩니다. (혹시 앱에 새 데이터가 안 보이면 아래 "🔄 manifest 버전 동기화"를 수동으로 눌러도 됩니다.)
새 시험(예: 주택관리사)은 manifest 포함 번들을 업로드하면 등록이 자동으로 됩니다(기존 시험 무수정, 새 시험·과목만 추가). 아래 "🗂 매니페스트 등록(새 시험)"은 수동 폴백용입니다.
🔍 검수 지적서 (수정 요청 → 데이터방 전달)
위에 로드한 기출 JSON을 게이트로 전수 검사한 위반 목록에, 직접 본 지적을 더해 복사·다운로드해서 데이터방에 넘깁니다. 지적된 항목만 수정되고 통과 문항은 안 건드립니다.
✏️ 시험 이름 바꾸기 (manifest)
내보내기·시험목록에 보이는 시험 이름만 고칩니다. 예: "주택관리사" → "주택관리사 1차". (문항·데이터는 안 건드림)
version +1 강제 갱신 (전체 사용자 캐시 갱신)
데이터(문항)는 그대로인데 version만 같아서 사용자들이 옛 데이터를 캐싱 중일 때 사용. 선택한 자격증의 bank·manifest version을 모두 +1 올려 모든 사용자가 새로 받게 합니다.

🖼️ 이미지 내보내기 (images → JSON 백업)
자격증 → 과목 → 회차 순으로 좁혀서 받을 수 있습니다. 과목·회차를 전체로 두면 그 위 범위 전부를 받습니다. (한국사는 과목이 1개라 회차로 좁히세요)
⚠️ 이미지가 많으면 파일이 큽니다(수십 MB+). 느리면 자격증별로 나눠 받으세요. 이 파일은 이미지 백업/이전용입니다.

파일명이 곧 키입니다 — 예: ksh74_q13_1.png → 키 ksh74_q13_1 → 문항에 img://ksh74_q13_1
업로드 시 가로 최대 1000px로 자동 리사이즈·압축해 images/{키} 문서에 저장합니다. 같은 키면 덮어씁니다.

'불러오기'를 누르면 업로드 이미지와 사용처를 스캔해 미사용 이미지를 표시합니다.
🗑️ 마스터 삭제 (안 쓰인 미사용 개념·표·암기·그래프 정리)
전 과목 문항 exp.cpt·개념의 tbl/mn/grp 링크와 대조해 어디에서도 참조하지 않는 미사용(고아) 마스터를 추립니다. 개념을 새로 만들어 옛 개념이 안 쓰이게 됐을 때 정리하기 좋습니다. 삭제 전 내보내기(JSON 백업)를 권장합니다. 사용 중 마스터는 일괄 삭제에서 자동 제외됩니다. (개념을 지우면 그 개념이 물던 표·암기가 새로 고아가 될 수 있으니, 삭제 후 다시 불러오기로 재스캔하세요.)
'불러오기'를 누르면 마스터와 사용처를 스캔해 미사용 마스터를 표시합니다.

🏷️ OX진술 태그 업로드

진술별 개념 태그(exp.ot)를 문항에 병합합니다. 과목당 파일 1개 — 형식: {cert, subject, ot:{문항id:[{cpt:[...]} | {skip:"..."}], ...}} (회차 다 합쳐서, set 불필요 · 문항 id로 찾음). 검증: ①ot 길이=exp.o 길이 ②참조 cpt가 개념 마스터에 존재(없으면 차단) ③문항 id 존재. 회차별 요약 표시.

🩹 콘텐츠 패치 업로드

해설·예시·띄어쓰기 부분 패치를 문항에 id로 콕 집어 병합합니다(문서 통째 덮어쓰기·재샤딩 없음). 형식: [{docId, id, o?, ex?, q?, opts?}, …](배열) 또는 {edits:[…]}. o→exp.o(선지 해설), ex→exp.ex(예시), q·opts→문두·보기. 검증: ①문항 id 존재 ②o 길이=현재 exp.o 길이 ③q·opts는 띄어쓰기만 다를 때만 통과(내용 바뀌면 자동 차단). 한 번에 파일 1개(콘텐츠 패치·띄어쓰기 패치 각각 드롭).
